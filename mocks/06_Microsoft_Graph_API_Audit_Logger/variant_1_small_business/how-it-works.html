üîí Security & Privacy

Your Data, Your Control

GraphAudit uses read-only Microsoft Graph API permissions. We never access file contents, email bodies, or message text‚Äîonly metadata (who, what, when, where).

Security Features:

  • Zero Data Retention: Logs stored in your own Azure Storage or S3 bucket. We don't store your data.
  • Encryption: All API calls use TLS 1.3. Logs encrypted at rest (AES-256).
  • Compliance: SOC 2 Type II certified. HIPAA, GDPR, CCPA compliant.
  • Access Control: SSO via Azure AD, Okta. Role-based permissions for viewing logs.
  • Audit Trail: We log who accessed audit logs (meta-logging for compliance).

Permissions Required: AuditLog.Read.All, Directory.Read.All, Mail.Read, Files.Read.All, User.Read.All (all read-only)

‚ùì Frequently Asked Questions

How is this different from Microsoft's built-in audit logs?

Microsoft 365 audit logs are only available for 90 days (or 1 year with E5 license). GraphAudit provides unlimited retention, custom alerts, SIEM integration, and user-friendly reports without requiring E5.

Do you store email content or file contents?

No. We only log metadata (who accessed what, when, and from where). We never access email bodies, file contents, or message text.

Can I export logs to my SIEM?

Yes. Professional and Enterprise plans include integrations with Splunk, Datadog, Azure Sentinel, and any SIEM that supports syslog or API ingestion.

What happens if I cancel?

You can export all logs before canceling. If you use your own storage (Azure/S3), logs remain accessible even after cancellation.

How does pricing work for growing teams?

We bill based on active user count (users who logged in within last 30 days). Pricing automatically adjusts as your team grows. No surprise overage fees.
